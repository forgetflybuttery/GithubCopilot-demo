# Skills Demo

## 说明
Skills 是定义在 `.github/skills/` 目录下的可复用自定义任务。通过 `/skill <名称> <参数>` 触发，让 Copilot 执行特定的标准化操作。

---

## 步骤演示

### 演示: 获取 GitHub 最近一周更新日志

1. 创建 `.github/skills/skill.md` 定义技能（本项目已创建）。
2. 在 Copilot 聊天输入:
   ```
   /skill github-changelog last-week
   ```
3. Copilot 从 `https://github.blog/changelog/` 抓取并格式化输出。
4. 生成 Markdown 文件保存到 `work/log/` 目录。

### 执行结果
技能执行后会在 `work/log/` 生成文件，例如：
```
work/log/github_changelog_20260315~20260322.md
```

---

## Skill 定义文件结构

`.github/skills/skill.md` 的定义:

```markdown
# GitHub Changelog Skill

## 描述
获取 GitHub 官方 changelog 中的重大更新，分类汇总 Actions 和 Copilot 更新。

## 参数
- `period`: 时间周期（例如 "last-week"）

## 实现
从 https://github.blog/changelog/ 获取条目，
每项更新控制在 150 字，输出为 Markdown。

## 输出
保存到 work/log/github_changelog_YYYYMMDD~YYYYMMDD.md
```

## 如何创建自定义 Skill

```bash
# 1. 创建 skills 目录
mkdir -p .github/skills

# 2. 创建 skill 定义文件
cat > .github/skills/deploy-check.md << 'EOF'
# Deploy Check Skill

## 描述
检查部署前的环境和依赖是否满足条件。

## 参数
- `env`: 目标环境 (staging/production)

## 实现
运行以下检查:
1. 检查必要环境变量
2. 验证 Bash 脚本语法
3. 确认目标服务器连通性
4. 输出检查报告
EOF

# 3. 使用
# /skill deploy-check staging
```

## 配套 Bash 脚本示例

以下是 Skill 实际调用的 Bash 脚本：

```bash
#!/bin/bash
# skill-runner.sh: 在 CI 中执行 skill 等效检查
set -euo pipefail

LOG_FILE="/tmp/skill_runner.log"
log_info()  { echo "[$(date +'%Y-%m-%d %H:%M:%S')] INFO:  $*" | tee -a "$LOG_FILE"; }
log_error() { echo "[$(date +'%Y-%m-%d %H:%M:%S')] ERROR: $*" | tee -a "$LOG_FILE" >&2; }

# 获取 changelog 并检查是否有 Actions 更新
fetch_changelog_summary() {
    local since_days=${1:-7}
    log_info "获取最近 ${since_days} 天的 GitHub changelog..."

    if ! command -v curl &>/dev/null; then
        log_error "curl 未安装，无法获取 changelog"
        return 1
    fi

    local url="https://github.blog/changelog/"
    local tmp_file="/tmp/changelog_raw.html"

    curl -sf "$url" -o "$tmp_file" --max-time 30 || {
        log_error "获取 changelog 失败"
        return 1
    }

    log_info "成功获取 changelog，开始解析..."

    # 检查是否有 Actions 更新
    if grep -qi "actions" "$tmp_file"; then
        log_info "发现 GitHub Actions 更新"
    fi

    # 检查是否有 Copilot 更新
    if grep -qi "copilot" "$tmp_file"; then
        log_info "发现 GitHub Copilot 更新"
    fi

    rm -f "$tmp_file"
    log_info "Changelog 摘要完成"
}

fetch_changelog_summary "${1:-7}"
```

## GitHub Actions Workflow 例子

```yaml
name: Skills Demo - Weekly Changelog Check

on:
  schedule:
    - cron: '0 9 * * 1'   # 每周一早上 9 点
  workflow_dispatch:

jobs:
  fetch-changelog:
    runs-on: ubuntu-latest

    steps:
      - name: Checkout code
        uses: actions/checkout@v4

      - name: Fetch GitHub changelog summary
        run: |
          echo "=== 获取本周 GitHub 更新 ==="
          curl -sf https://github.blog/changelog/ \
            | grep -oP '(?<=<h3>)[^<]+' \
            | head -20 \
            | while IFS= read -r line; do
                echo "- $line"
              done

      - name: Save report
        run: |
          mkdir -p work/log
          DATE=$(date +'%Y%m%d')
          REPORT="work/log/github_changelog_${DATE}.txt"
          echo "# GitHub Changelog Report - $(date)" > "$REPORT"
          echo "Generated by GitHub Actions" >> "$REPORT"
          echo "Report saved to $REPORT"

      - name: Upload changelog report
        uses: actions/upload-artifact@v4
        with:
          name: changelog-report
          path: work/log/
```

## Skills vs Instruction 对比

| 特性 | Skills | Instruction |
|------|--------|-------------|
| 触发方式 | `/skill <名称>` 显式调用 | 自动应用于所有对话 |
| 适用场景 | 特定的一次性任务 | 全局行为规范 |
| 参数支持 | 支持动态参数 | 静态规则 |
| 典型用途 | 获取数据、执行检查 | 代码风格、角色设定 |
